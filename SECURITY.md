# Security Policy

## 适用范围

本项目为纯文档型仓库（研究方法论 + 案例分析），不包含可执行代码、API 端点或数据处理逻辑。

## 报告方式

如发现以下问题，请通过 GitHub Issues 报告：

- 文档中的事实错误或数据引用错误
- 许可证合规问题
- 贡献者个人信息泄露

## 不支持的安全问题

本仓库不接受以下类别的报告：

- 代码漏洞（无任何可执行代码）
- 依赖项安全（无依赖管理文件）
- API/基础设施安全（无部署的服务端组件）
