# Contributing

感谢你对 SCSPF 框架的关注。本项目欢迎以下形式的贡献：

## 贡献方式

- **案例提交**：在 `cases/` 目录下按 METHODOLOGY 第 10 节格式提交新比赛分析
- **方法论改进**：对框架层的因子修正、新变量建议，通过 Issue 讨论后提交 PR
- **文档勘误**：对 README、METHODOLOGY、案例中的错误或过时描述的修正
- **数据源补充**：第 8 节数据因子清单中新增可靠数据来源

## 提交流程

1. Fork 本仓库
2. 为你的改动创建分支（如 `case/2030-xxx` 或 `fix/methodology-typo`）
3. 提交时使用语义化 commit message（如 `docs: fix typo in Layer 3`）
4. 发起 Pull Request，描述改动内容和依据

## 案例格式

新建案例分析请参照 `cases/2026-world-cup-3rd-place-france-england.md` 的结构，至少包含：

- 比赛信息表
- 逐层分析（前置层 + ①→⑤ 层）
- 综合结论与模型校准反思

## 行为准则

本项目遵循基本的开源礼仪：尊重不同观点，聚焦方法论讨论而非人身攻击。
